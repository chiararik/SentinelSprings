# SentinelSprings_Trends

**Per-pixel vegetation trend analysis from Sentinel-2, on Google Earth Engine.**
Mann-Kendall and Sen's slope on seasonal 90th-percentile composites of NDVI and NDMI, 2017&ndash;today.

## What this tool does

`SentinelSprings_Trends.js` is a standalone Google Earth Engine (GEE) script that:

1. Loads a user-defined polygon **Area of Interest** (AOI) and an **active vegetation season** (day-of-year window).
2. Builds the full Sentinel-2 SR Harmonized archive over the AOI within that season, masks clouds and shadows with **Cloud Score+**, and computes the requested vegetation indices (**NDVI** and/or **NDMI**).
3. Reduces each season to an **annual 90th-percentile composite** per pixel per index.
4. Applies the **Mann-Kendall** trend test and the **Sen's slope** estimator independently to each annual time series.
5. Exports a multi-band GeoTIFF (one file per index) with `sens_slope`, `sens_slope_significant`, `kendall_tau`, `z`, `p_value`, `S`, and `valid_years`, plus one annual p90 composite per year per index.

The tool is designed for **multi-partner deployment in the SentinelSprings consortium**: each partner runs the same pipeline on their own AOI, varying only a handful of local parameters, so trend products generated by different partners are methodologically comparable.

## Project context

This script is developed as part of [**SentinelSprings**](https://sentinelsprings.eu), an EU Horizon Europe / Water4All project (Grant Agreement n&deg;101060874) that positions springs as early-warning sentinels of climate and anthropogenic pressures on groundwater-dependent ecosystems. The remote-sensing component characterises the vegetation in the catchments and surroundings of the pilot springs across six European sites.

## Quick start

1. Open the [Google Earth Engine Code Editor](https://code.earthengine.google.com/).
2. Paste the contents of `SentinelSprings_Trends.js` into a new script.
3. In **Section&nbsp;1 &mdash; User parameters** (the only block you need to edit), set:
   - `aoi` &mdash; your AOI asset path (a `FeatureCollection` of polygons or multipolygons).
   - `aoiLabelProperty` &mdash; the column of your shapefile to use as feature label (e.g. `'name'`, `'CODICE'`).
   - `indices` &mdash; one or more of `'NDVI'`, `'NDMI'`.
   - `doyStart`, `doyEnd` &mdash; the active vegetation season of your AOI, in day-of-year.
   - `startYear`, `endYear` &mdash; the analysis period (2018+ recommended).
   - `driveFolder` &mdash; the destination folder in your Google Drive.
4. Click **Run**. Inspect the console output: AOI summary, per-feature time-series chart, coverage diagnostic, and area statistics.
5. Open the **Tasks** panel (top-right of the Code Editor) and click **RUN** on each export task. GeoTIFFs will appear in the specified Drive folder.

## Parameters

| Parameter | Role | Default | Notes |
|---|---|---|---|
| `aoi` | Area of Interest | &mdash; | Polygon or MultiPolygon `FeatureCollection`. |
| `aoiLabelProperty` | Feature label column | `null` | Falls back to `system:index` if `null`. |
| `indices` | Vegetation indices to compute | `['NDVI']` | Any subset of `'NDVI'`, `'NDMI'`. |
| `doyStart`, `doyEnd` | Active vegetation season | `152, 273` | Day-of-year (1 Jun &ndash; 30 Sep). |
| `startYear`, `endYear` | Analysis period | `2018, 2024` | S2 SR begins on 2017-03-28. |
| `csThreshold` | Cloud Score+ threshold | `0.60` | Pixels with `cs < threshold` are masked. |
| `minImgsPerSeason` | Min scenes per pixel-year | `5` | Pixel-years with fewer obs. are masked. |
| `minValidYears` | Min annual composites per pixel | `6` | Pixels below this are masked from the trend output. |
| `pSignificance` | Significance level | `0.05` | Used for the `sens_slope_significant` band and area stats. |
| `chartYMin`, `chartYMax` | Chart Y-axis range | `null, null` | Both `null` lets Google Charts auto-scale. |
| `exportTo` | Export destination | `'drive'` | One of `'drive'`, `'asset'`, `'both'`. |
| `driveFolder` | Google Drive folder | `'SentinelSprings_GEE_exports'` | Auto-created if missing. |
| `assetFolder` | GEE asset folder | `'users/your_username/SentinelSprings'` | Must already exist. |
| `exportCRS` | Output CRS | `null` | `null` &rarr; native S2 UTM, square 10&nbsp;m pixels. |

## Outputs

For each requested index (NDVI and/or NDMI), the script queues:

- **One trend raster GeoTIFF** &mdash; multi-band, 10&nbsp;m, in native S2 UTM by default. Bands:

  | Band | Description |
  |---|---|
  | `sens_slope` | Sen's slope, in index units per year (all pixels with enough data). |
  | `sens_slope_significant` | Sen's slope masked to pixels with `p_value <= pSignificance`; non-significant pixels are no-data. |
  | `kendall_tau` | Kendall's &tau;, in [&minus;1, +1]. |
  | `z` | Standardised Mann-Kendall statistic. |
  | `p_value` | Two-sided asymptotic p-value. |
  | `S` | Mann-Kendall S statistic. |
  | `valid_years` | Per-pixel number of valid annual composites. |

- **One annual p90 composite GeoTIFF per year** &mdash; single-band, 10&nbsp;m.

In the Code Editor:
- A **time-series chart** per index, one line per feature in the AOI shapefile.
- A **coverage diagnostic** printout (min / max / mean / median of `valid_years` over the AOI).
- An **area summary** per index: hectares with significant positive / negative trend at the chosen p-threshold.

## Methodology summary

- **Reflectance**: Sentinel-2 Level&nbsp;2A Surface Reflectance, harmonised collection (`COPERNICUS/S2_SR_HARMONIZED`).
- **Cloud and shadow mask**: Cloud Score+ (`GOOGLE/CLOUD_SCORE_PLUS/V1/S2_HARMONIZED`), threshold `cs >= 0.60`.
- **Indices**:
  - NDVI = (B8 &minus; B4) / (B8 + B4) &nbsp;&mdash;&nbsp; greenness, 10&nbsp;m.
  - NDMI = (B8 &minus; B11) / (B8 + B11) &nbsp;&mdash;&nbsp; canopy water content, 20&nbsp;m (resampled).
- **Compositing**: per-pixel 90th percentile of the seasonal stack &mdash; robust to residual cloud and snow, independent of season length.
- **Trend test**: Mann-Kendall (Mann 1945; Kendall 1975), variance with continuity correction, two-sided asymptotic p-value (Zelen & Severo 1964 normal-CDF approximation).
- **Slope estimator**: Sen's slope (Sen 1968), median of all pairwise slopes between annual composites.
- **Joint NDVI / NDMI reading** is recommended whenever both are computed: divergence between the two indices (e.g. stable NDVI with declining NDMI) flags early-warning patterns of water stress invisible to NDVI alone.

Full theoretical details, equations, and recommended consortium-level parameter harmonisation are documented in the companion **ATBD** (Algorithm Theoretical Basis Document) &mdash; see [Documentation](#documentation).

## Known limitations

- **Statistical power**: with Sentinel-2 available from 2017, the per-pixel sample size is n &asymp; 7&ndash;9. Mann-Kendall has limited power at this n; only strong and consistent trends reach significance. Non-significant should be read as *no evidence of trend*, not *evidence of no trend*.
- **Asymptotic p-value**: the normal-CDF approximation is well-established for n &gt; 10 and slightly biased for shorter series. Exact MK tables (Hipel & McLeod 1994) can be applied in post-processing.
- **Influence of anomalous years**: with small n, single extreme years (e.g. the 2022 European drought) can drive the Sen's slope. A sensitivity check with the anomalous year excluded is recommended.
- **p90 stability with few observations**: when the per-pixel observation count is below &asymp;&nbsp;5, the p90 collapses to the seasonal maximum and becomes noisier.
